# Scene Teaching（终端版）

综合 `slide-content` 与 `slide-actions` 哲学，在终端生成**可朗读、可互动**的场景脚本。

## 幻灯片内容哲学

**讲义是视觉辅助，不是逐字稿。**

- 幻灯片/on-slide 区：关键词、短语、要点（每条 ≤20 词或 ≤30 汉字）
- 讲解区：完整口语化讲解、过渡、鼓励、举例
- 禁止在 slide 区写对话式长句或署名「老师寄语」

## 终端输出结构

每个场景输出 Markdown 文件，结构如下：

```markdown
# Scene {order}: {title}

> {description}

## 要点（Slide View）
- 要点 1
- 要点 2

## 讲解脚本

### [Teacher · {name}]
{自然口语讲解，2-3 短句}

### [Assistant · {name}]（可选）
{补充一句，≤80 字}

## 白板（可选）
```
{ASCII 或 Mermaid 图示 — 公式、流程、表格}
```

## 讨论钩子（可选，每课最多 1-2 次）
- **话题**: ...
- **引导**: ...
- **首发**: {student agent name}

## 过渡
{连接下一场景的 1 句过渡语}
```

## 同课连贯性（CRITICAL）

- **第一场景**：问候 + 课程介绍（唯一可问候的场景）
- **中间场景**：「接下来我们看…」「刚才提到的…」— 禁止重新自我介绍
- **最后场景**：总结 + 收束
- 引用前文用「刚才我们讲到」— 不说「上节课」

## 讲解长度（按角色）

| 角色 | 单次讲解总字数 |
|------|--------------|
| teacher | ~100 字（2-3 短句） |
| assistant/evaluator/facilitator | ~80 字 |
| student | ~50 字（1-2 句） |

## 互动设计

- 每 slide 场景 3-6 个讲解段落
- 复杂概念优先用 ASCII/Mermaid 白板辅助
- 开放性问题场景末尾加「讨论钩子」
- quiz 场景改用 `quiz-generator.md`

## 语言

严格遵循 `languageDirective`。
